# 代码清理与迭代计划模板

## 优先级

- [ ] **P0**：需立即清理（安全风险、显著成本、阻塞其他工作）
- [ ] **P1**：当前迭代内清理
- [ ] **P2**：放入 Backlog / 下一迭代

---

## 可立即安全删除

### 条目：[名称/描述]

| 字段         | 详情                                        |
| ------------ | ------------------------------------------- |
| **位置**     | `path/to/file.ts:line`                      |
| **原因**     | 为什么应该删除                              |
| **依据**     | 无引用、已关闭的 feature flag、已废弃的 API |
| **影响**     | 无 / 低 - 无活跃消费者                      |
| **删除步骤** | 1. 删除代码 2. 删除测试 3. 删除配置         |
| **验证**     | 运行测试、检查无运行时错误、监控日志        |

---

## 延期删除（需制定计划）

### 条目：[名称/描述]

| 字段           | 详情                                           |
| -------------- | ---------------------------------------------- |
| **位置**       | `path/to/file.ts:line`                         |
| **延期原因**   | 存在活跃消费者、需要迁移、需干系人确认         |
| **前置条件**   | Feature flag 关闭 2 周、遥测数据显示使用量为 0 |
| **破坏性变更** | 列出所有 API/契约变更                          |
| **迁移计划**   | 消费者迁移步骤                                 |
| **时间线**     | 目标日期或迭代                                 |
| **负责人**     | 负责的个人/团队                                |
| **验证指标**   | 确认可安全删除的指标（错误率、使用量）         |
| **回滚方案**   | 发现问题时如何恢复                             |

---

## 删除前检查清单

- [ ] 在代码库中搜索了所有引用（`rg`、`grep`）
- [ ] 检查了动态调用/反射调用的使用
- [ ] 确认无外部消费者（API、SDK、文档）
- [ ] 审查了 feature flag 遥测数据（如适用）
- [ ] 测试已更新/删除
- [ ] 文档已更新
- [ ] 已通知团队（如为共享代码）
