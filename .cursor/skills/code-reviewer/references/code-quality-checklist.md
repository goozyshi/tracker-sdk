# 代码质量检查清单

## 错误处理

### 需标记的反模式

- **异常吞没**：空的 catch 块或仅打印日志的 catch
  ```javascript
  try { ... } catch (e) { }  // 静默失败
  try { ... } catch (e) { console.log(e) }  // 打印后遗忘
  ```
- **过于宽泛的 catch**：捕获 `Exception`/`Error` 基类而非具体类型
- **错误信息泄露**：堆栈信息或内部细节暴露给用户
- **缺失错误处理**：可失败操作（I/O、网络、解析）未使用 try-catch
- **异步错误处理**：未处理的 Promise rejection、缺失 `.catch()`、无 Error Boundary
- **竞态请求**：快速操作触发多个并行请求，旧响应晚于新响应到达导致数据被覆盖

### 最佳实践检查项

- [ ] 错误在合适的边界被捕获
- [ ] 错误消息对用户友好（不暴露内部细节）
- [ ] 错误日志包含足够的调试上下文
- [ ] 异步错误被正确传播或处理
- [ ] 可恢复错误定义了降级行为
- [ ] 关键错误触发告警/监控

### 自问

- "当这个操作失败时会发生什么？"
- "调用方能否感知到出了问题？"
- "是否有足够的上下文来调试这个错误？"

## 性能与缓存

### CPU 密集操作

- **热点路径中的高开销操作**：循环中的正则编译、JSON 解析、加密运算
- **阻塞主线程**：同步 I/O、未使用 Worker/异步的重计算
- **不必要的重复计算**：相同计算被执行多次
- **缺失 memoization**：纯函数以相同输入被反复调用

### 数据库与 I/O

- **N+1 查询**：循环中逐条查询而非批量处理
  ```javascript
  // 反例：N+1
  for (const id of ids) {
    const user = await db.query(`SELECT * FROM users WHERE id = ?`, id)
  }
  // 正例：批量查询
  const users = await db.query(`SELECT * FROM users WHERE id IN (?)`, ids)
  ```
- **缺失索引**：查询未建立索引的列
- **过度获取**：仅需少量字段时使用 SELECT *
- **无分页**：将整个数据集加载到内存

### 缓存问题

- **高开销操作未使用缓存**：重复的 API 调用、DB 查询、计算
- **缓存无 TTL**：过期数据被无限期返回
- **缓存无失效策略**：数据更新后缓存未清除
- **缓存 Key 冲突**：Key 唯一性不足
- **用户级数据被全局缓存**：安全/隐私问题

### 内存

- **无限增长的集合**：Array/Map 无大小限制地持续增长
- **大对象滞留**：持有引用导致 GC 无法回收
- **循环中拼接字符串**：应使用 StringBuilder/join
- **整体加载大文件**：应使用流式处理
- **组件销毁后未清理副作用**：异步回调、事件监听、定时器、订阅在组件卸载后仍在执行

### 自问

- "这个操作的时间复杂度是多少？"
- "数据量增长 10 倍/100 倍时表现如何？"
- "这个结果是否可缓存？是否应该缓存？"
- "是否可以批量处理而非逐条执行？"

## 边界条件

### Null/Undefined 处理

- **缺失 null 检查**：访问可能为 null 的对象属性
- **Truthy/Falsy 混淆**：当 `0` 或 `""` 是合法值时使用 `if (value)`
- **Optional Chaining 滥用**：`a?.b?.c?.d` 掩盖了结构性问题
- **Null vs Undefined 不一致**：混用且无明确约定

### 空集合

- **未处理空数组**：代码假设数组有元素
- **空对象边界情况**：对空对象使用 `for...in` 或 `Object.keys`
- **首/末元素访问**：`arr[0]` 或 `arr[arr.length-1]` 未检查长度

### 数值边界

- **除零错误**：除法前未检查除数
- **整数溢出**：大数超出安全整数范围
- **浮点数比较**：使用 `===` 而非 epsilon 比较
- **负值**：索引或计数不应为负
- **Off-by-one 错误**：循环边界、数组切片、分页

### 字符串边界

- **空字符串**：未作为边界情况处理
- **纯空白字符串**：通过 truthy 检查但实际为空
- **超长字符串**：无长度限制导致内存/显示问题
- **Unicode 边界情况**：Emoji、RTL 文本、组合字符

### 需标记的常见模式

```javascript
// 危险：无 null 检查
const name = user.profile.name

// 危险：无长度检查直接取元素
const first = items[0]

// 危险：无除零检查
const avg = total / count

// 危险：truthy 检查排除了合法值
if (value) { ... }  // 0、""、false 均不通过
```
